# Domain 8: 3D, Depth, and Geometry

This is a more niche domain, but it is real and worth mentioning.

## What This Domain Is

This domain includes:

- depth estimation
- image-to-3D workflows
- text-to-3D workflows
- geometry-aware models
- point-cloud style outputs

## Why You Might Choose It

- it shows that deep learning is broader than the most common demos
- some interesting pretrained models already exist here
- it can lead to unusual and memorable projects

## Example Applications

- estimate scene depth from an image
- generate a 3D object from an image or prompt
- build a workflow around 3D model generation
- explore geometry-aware outputs in a small demo

## Which Technical Paths Fit Best

- Path 2 if you want to use pretrained models
- Path 3 if you want to adapt a model or workflow
- this domain is usually not the best place to start with full custom training in a short course

## Recommended Frameworks to Start With

- Hugging Face models for depth or 3D tasks
- PyTorch
- Gradio or another custom frontend for visualization

## Official Docs and Starting Points

- Hugging Face models: <https://huggingface.co/models>
- Hugging Face tasks: <https://huggingface.co/tasks>
- PyTorch: <https://pytorch.org/>
- Gradio: <https://www.gradio.app/docs>

## Good Advice Before You Begin

- keep the scope small
- focus on one clear input/output workflow
- prefer pretrained models over ambitious custom training
