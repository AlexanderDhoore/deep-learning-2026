# Application Domains

Use this file after `01-technical-paths.md`.

The technical path tells you **how** you want to work with models.
The application domain tells you **what kind of data or problem** you want to work on.

This file is only a high-level map.
If one domain looks interesting, open the matching file in `domains/` for more detail.

## Domain 1: Computer Vision

This domain is about images and video.

Typical tasks:

- image classification
- object detection
- segmentation
- depth estimation
- keypoint detection
- visual inspection

Good fit if you want:

- something visual and easy to demo
- a project with strong industrial relevance
- a domain with many good pretrained models and practical frameworks

Read next:

- [`domains/01-computer-vision.md`](domains/01-computer-vision.md)

## Domain 2: Language Models and LLM Apps

This domain is about text-based AI systems.

Typical tasks:

- chatbots
- question answering
- summarization
- rewriting
- extraction
- tool calling

Good fit if you want:

- to build assistants, chat systems, or writing tools
- to work with pretrained or adapted models
- to prototype quickly around a clear user interaction

Read next:

- [`domains/02-language-models-and-llm-apps.md`](domains/02-language-models-and-llm-apps.md)

## Domain 3: Multimodal and Document AI

This domain is about systems that combine multiple kinds of input or output, such as text, images, audio, or documents.

Typical tasks:

- visual question answering
- image captioning
- document understanding
- OCR-enhanced workflows
- mixed-media assistants

Good fit if you want:

- something that feels modern and impressive
- a project that combines text with images, audio, or documents
- a workflow built around richer user input

Read next:

- [`domains/03-multimodal-and-document-ai.md`](domains/03-multimodal-and-document-ai.md)

## Domain 4: Semantic Search and Retrieval

This domain is about finding information by meaning instead of only by exact keyword matching.

Typical tasks:

- semantic search
- retrieval
- similarity matching
- retrieval-augmented generation

Good fit if you want:

- to build systems that work on real documents or knowledge bases
- a practical AI workflow with clear product value
- a project that uses embeddings and search instead of only generation

Read next:

- [`domains/04-semantic-search-and-retrieval.md`](domains/04-semantic-search-and-retrieval.md)

## Domain 5: Recommendation Systems

This domain is about suggesting useful items to a user.

Typical tasks:

- recommendation
- ranking
- similarity-based suggestion
- personalized selection

Good fit if you want:

- a product-oriented project
- to build around user preferences or item similarity
- a strong demo with a clear user experience

Read next:

- [`domains/05-recommendation-systems.md`](domains/05-recommendation-systems.md)

## Domain 6: Speech and Audio

This domain is about systems that understand audio or generate audio.

Typical tasks:

- speech-to-text
- text-to-speech
- audio classification
- music generation
- voice adaptation

Good fit if you want:

- an interactive demo with sound or voice
- something that combines well with language models
- a project where the input and output feel immediately understandable

Read next:

- [`domains/06-speech-and-audio.md`](domains/06-speech-and-audio.md)

## Domain 7: Time-Series and Sensor Signals

This domain is about data measured over time.

Typical tasks:

- forecasting
- anomaly detection
- condition monitoring
- signal classification

Good fit if you want:

- something closer to sensors, devices, or engineering
- a more industrial or electronics-ICT-flavored project
- a project built around measurements, signals, or monitoring

Read next:

- [`domains/07-time-series-and-sensor-signals.md`](domains/07-time-series-and-sensor-signals.md)

## Domain 8: 3D, Depth, and Geometry

This domain is about spatial reasoning, depth, geometry, and 3D outputs.

Typical tasks:

- depth estimation
- image-to-3D
- text-to-3D
- geometry-related model workflows

Good fit if you want:

- something more unusual than the standard domains
- a niche but memorable project
- to explore pretrained models around 3D or depth

Read next:

- [`domains/08-3d-depth-and-geometry.md`](domains/08-3d-depth-and-geometry.md)

## Domain 9: Reinforcement Learning and Control

This is a slightly different kind of choice.

It is less about a data modality and more about a different learning setup: an agent interacts with an environment, receives rewards, and improves over time.

Good fit if you want:

- games
- control
- simulation
- a project where learning by interaction is the main idea

Read next:

- [`domains/09-reinforcement-learning-and-control.md`](domains/09-reinforcement-learning-and-control.md)

## Domain 10: Edge AI and Deployment

This is also a slightly different kind of choice.

It is less about the type of data and more about where the model runs and how well it runs there.

Good fit if you want:

- embedded systems
- deployment
- runtime optimization
- real hardware constraints

Read next:

- [`domains/10-edge-ai-and-deployment.md`](domains/10-edge-ai-and-deployment.md)

## How To Choose

Choose a domain that feels:

- interesting to you
- feasible in the available time
- easy enough to demo clearly
- large enough to show real technical work

Then open the matching file in `domains/`, choose a framework stack, and start building a small first prototype.
