# Assignment Brief

Use this page as the practical brief for your project.

## How This Part of the Course Works

- The course has 4 lessons of 4 hours, plus the exam.
- The first lesson already introduced you to practical generative AI.
- The remaining lessons are mainly for building your own project.
- The goal now is to choose a good path, choose a good domain, and build something real.

## Goal

Build a unique AI application and present it at the exam.

## How You Will Be Evaluated

- You will build a unique AI application or prototype.
- At the exam, you will present what you built.
- You will explain your design choices, framework stack, and code.
- You may use AI tools while building, but during the exam you must understand what your system does.

## Expected Output

You should build:

- one working application or prototype
- with a clear AI component
- and a clear user-facing interface or interaction flow
- based on one main AI framework or model stack that you learn in depth

The interface may be:

- a web app
- an API with a small frontend
- a local GUI
- a command-line application with a strong demo
- an edge-device demo

## Important Rule

You may use AI tools while building.

But during the exam, you must be able to explain:

- your architecture
- your code
- your framework choices
- your data flow
- your model choice
- your limitations and future improvements

## A Good Project Formula

> input -> AI model -> useful output -> interface

Examples:

- image -> detector -> labels and boxes -> web app
- user question + documents -> retrieval + LLM -> answer -> chat UI
- speech -> transcription -> summary -> web app
- sensor data -> anomaly detector -> warning -> dashboard

## Good Project Characteristics

- unique topic
- feasible within the course time
- clear technical challenge
- easy to demonstrate
- enough code and technical reasoning to defend orally
- one main framework choice that you can explain well
- not overly ambitious in both model-building and deployment at the same time

## Scope Matters

- Pick something interesting, but also feasible.
- A good project usually has one clear problem, one main framework, and one demoable result.
- Avoid choosing the hardest model-training path and the hardest deployment path at the same time.

## What I Want to See

- a unique project
- a clear technical path
- one main framework stack
- visible effort
- a working demo or prototype
- understanding of the code and architecture

## Suggested Submission Shape

- source code in Git
- short README
- short demo
- presentation and oral defense during the exam

## What To Do Next

- choose a technical path
- choose an application domain or framework stack
- then choose the matching domain or framework stack
- sketch a project idea
- start building

## Final Message

- Deep learning is a very broad field.
- You do not need to master all of it in this course.
- The goal is to leave this course with a better map of the field.
- The goal is to leave this course with experience building one real AI system.
- The goal is to leave this course with confidence that you can learn a new framework on your own.
